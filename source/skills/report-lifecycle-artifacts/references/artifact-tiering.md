# Lifecycle Artifact Tiering

Use the smallest tier that satisfies the user's explicit artifact request.

| Tier | Use When | Typical Outputs |
| --- | --- | --- |
| 0 | Small fixes or one-shot tasks | No lifecycle package; use ordinary final reporting. |
| 1 | Single feature or requirement slice | Requirements contract, acceptance criteria, validation notes. |
| 2 | Module, subsystem, or release slice | Requirements, WBS, delivery architecture package, implementation design record, validation/QA plan. |
| 3 | Product/release lifecycle package | Full artifact pack from discovery through release and retrospective. |

Full lifecycle packages require explicit artifact intent.
