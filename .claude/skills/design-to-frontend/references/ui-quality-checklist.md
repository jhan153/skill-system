# UI Quality Checklist

Use this checklist before reporting a design-to-frontend task complete when visual quality matters.

## Check order

1. Information hierarchy: primary information and primary action are clear.
2. Layout: regions align, spacing is intentional, and responsive order is stable.
3. Components: repeated patterns reuse existing components or a justified local boundary.
4. States: visible states and required interaction states are implemented or reported as gaps.
5. Tokens: colors, typography, spacing, radius, and elevation use repo conventions or documented substitutions.
6. Accessibility: labels, focus, keyboard, target size, contrast, and text overflow are checked where possible.
7. Visual polish: density, alignment, rhythm, media framing, icons, and text fitting are checked.

## Report discipline

- Mark unavailable design context, private assets, proprietary fonts, and inferred measurements.
- Do not claim visual completion from build success alone.
- Keep subjective polish notes separate from standards-backed accessibility findings.
