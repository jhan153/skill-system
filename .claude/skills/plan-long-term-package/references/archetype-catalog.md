# Archetype Catalog

## Selection Rule

```yaml
package_selection:
  archetype: "primary work type"
  modifiers:
    - "additional validation condition"
```

- `archetype`: primary nature of the work.
- `modifier`: extra risk condition that strengthens validation.
- `required contract docs`: docs generated by the init script and checked by the validator.
- Universal docs: every archetype also generates `release-gate` and `agent-handoff-index`, even when the table below lists only domain-specific contracts.

## Core Archetypes

| --- | --- | --- | --- |

| `web-product` | Browser-based web product | capability map, route contract, UI state contract, API contract, theme token contract, release gate | Admin web dashboard |
| `backend-service` | Server, API, DB, domain logic, or queue work | capability map, API contract, data contract, integration contract, observability contract, release gate | Order API server |

## Modifiers

| Modifier | Meaning | Additional Contract Docs |
| --- | --- | --- |
| `no-behavior-change` | Feature behavior must not change | behavior parity contract |
| `strict-behavior-parity` | Existing behavior must be validated as equivalent | behavior parity contract, parity contract |
| `legacy-parity` | Legacy output must remain equivalent | old-new mapping, parity contract, compatibility matrix |
| `performance-critical` | Performance must be verified numerically | performance budget, benchmark contract |
| `realtime-critical` | Latency, frame time, and loop timing matter | runtime loop contract, performance budget |
| `rendering-heavy` | Rendering pipeline, GPU resources, or visual regression matter | rendering pipeline contract, resource lifecycle contract, visual regression gate |
| `local-device-runtime` | Local storage, permissions, lifecycle, and device resources matter | app runtime contract, platform lifecycle contract, local storage contract |
| `hardware-integration` | Pin map, power, signal, communication, and hardware revisions matter | hardware interface contract, pin map, protocol contract |
| `mesh-processing-heavy` | Mesh cut, patch, replacement, geometry validity, or scanbody-like mesh ops matter | algorithm inventory, mesh ops contract, tolerance contract, visual regression gate, gap registry |
| `gap-tracking` | Missing capabilities must be explicitly tracked to closure | gap registry |
| `external-api` | External API failure, rate limit, and fallback matter | integration contract, failure matrix, fallback contract |
| `rollback-required` | Rollback conditions and procedure are mandatory | rollback plan, rollback trigger |
| `cross-session-handoff` | Later sessions or agents must continue from docs alone | source-of-truth policy, handoff index |
| `public-api-sensitive` | Breaking change, versioning, and compatibility matter | public API contract, compatibility policy, versioning policy, deprecation policy |
| `data-sensitive` | Data loss, migration, privacy, or schema change matters | data contract, migration map, compatibility matrix |
| `security-sensitive` | Auth, permission, secret, or attack surface review matters | security contract |

## Selection Decision Table

| If the task says... | Choose | Add modifier |
| --- | --- | --- |
| Change architecture rules, layers, dependency direction, or state ownership while preserving behavior | `architecture-refactor` | `no-behavior-change`, `strict-behavior-parity`, `rollback-required` |
| Split a large class/module or reduce coupling without changing architecture style | `large-scale-refactor` | `no-behavior-change`, `strict-behavior-parity` |
| Split a codebase into packages/modules with public boundaries | `modularization` | `public-api-sensitive` |
| Move platform, build system, storage, framework, or repo foundation | `migration-modernization` | `legacy-parity`, `rollback-required`, `data-sensitive` |
| Build a local device app with lifecycle, storage, packaging, and permissions | `application-product` | `local-device-runtime` |
| Build a browser app with routes, API boundary, and responsive UI | `web-product` | `external-api` |
| Replace geometry, optimization, ranking, or compute output logic | `algorithm-engine` | `strict-behavior-parity`, `performance-critical` |
| Build a realtime camera/sensor/simulation loop | `realtime-runtime-pipeline` | `realtime-critical`, `performance-critical` |
| Build rendering pipeline, scene graph, GPU resources, or shaders | `rendering-engine` | `rendering-heavy`, `performance-critical` |
| Build a product that includes heavy 3D rendering but also app lifecycle/UI | `application-product` | `rendering-heavy`, `performance-critical`, `local-device-runtime` |
| Build a product or engine with mesh cut/fill/replacement operations | `application-product` or `algorithm-engine` | `mesh-processing-heavy`, `strict-behavior-parity` |
| Integrate external API, SDK, device, file format, or cloud service | `integration-adapter` | `external-api`, `rollback-required` |

## Skill Rules

- Archetype determines the primary package type.
- Modifiers add validation conditions and required contract docs.
- Required docs are the union of archetype docs and modifier docs.
- `release-gate` and `agent-handoff-index` are universal docs added to every archetype.
- Validator must check the same union generated by init.
- Default validation checks package structure and source-of-truth consistency.
- Strict validation checks release-critical semantic content: numeric thresholds, budgets, tolerances, benchmark rows, evidence artifacts, and non-placeholder contract rows.
- Quality lint checks handoff readability: group relevant specs, dependency metadata, structured acceptance criteria, vague canonical IDs, and compact quickstart.
- `architecture-refactor` and `large-scale-refactor` stay separate.
- `application-product` and `web-product` stay separate.
- Application products include local device runtime, lifecycle, storage, permission, rendering/engine concerns, and packaging.
- Web products focus on routes, browser state, API boundary, responsive UI, theme tokens, and deployment boundary.
