# Section Page Checklist

Use this checklist when implementing or reviewing a section-based web page.

## Sections

- Hero or first viewport.
- Proof, product reveal, or object/place/person signal.
- Feature, workflow, comparison, or detail sections.
- Supporting content such as specs, social proof, FAQ, or docs.
- Final CTA or next action.

## Constraints

- Do not create a landing page when the user asked for an app/tool first screen.
- The first viewport should make the brand/product/place/object/person or literal offer clear.
- Section order should remain coherent on mobile and desktop.
- Text must fit without overlap, clipping, or button overflow.
- Media should reveal the relevant subject rather than act as vague atmosphere.
- Use semantic sections, headings, links, and buttons.

## Evidence

- First viewport screenshot.
- Mobile and desktop responsive order check.
- Heading/landmark/CTA accessible-name check.
- Explicit note for missing assets, missing copy, or inferred section order.
